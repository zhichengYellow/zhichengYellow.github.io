  • **tags (标签)**：像关键词，一篇文章可以有多个（比如：Java, 笔记）。
  • **categories (分类)**：像文件夹，支持多级（比如：后端技术 -> Java）。

四、 常用指令速查表

简写指令 完整指令 作用 备注
hexo n "..." hexo new "..." 新建文章 生成在 source/_posts
hexo s hexo server 启动本地预览 访问 localhost:4000
hexo g hexo generate 生成网页 编译 Markdown 为 HTML
hexo d hexo deploy 部署/发布 推送到 GitHub
hexo clean hexo clean 清理缓存 遇到玄学问题先执行它
hexo new page "..." hexo new page "..." 新建独立页面 如“关于我”、“友情链接”

五、 如果我想换电脑写博客怎么办？

这是很多新手容易踩的坑。GitHub 仓库 (yellow386.github.io) 里存的只是生成的网页 (HTML)，不是你的源代码 (Markdown)！

为了防止电脑坏了导致博客源码丢失，建议你做最后一件重要的事：

  1. 在 GitHub 上新建一个仓库，叫 hexo-blog-source (名字随意，设为 Private 私有仓库最好)。
  2. 把你的 s 文件夹整个上传到这个新仓库里备份。
  3. 以后每次写完，除了 hexo d 发布网页外，记得也要把源码 git push 到这个私有仓库备份。
